# LASER 回环约束坐标语义修复设计

## 1. 背景

当前回环分支已经完成以下改动：

- 第一阶段按自然排序后的统一图像清单运行；
- 相邻窗口配准统一保留置信度最高的前 `30%`；
- 每个窗口缓存第一阶段绝对变换 `sim3_abs` 和相对顺序边
  `sim3_edge`；
- 第一阶段立即将绝对 Sim(3) 应用到窗口相机位姿；
- 全局优化后只向缓存应用优化前后的增量。

KITTI 00 的云端对照实验使用了同一批第一阶段缓存。跳过回环优化时，
ATE RMSE 为 `18.323618`；加入 20 条回环约束后，ATE RMSE 增加到
`30.154020`，平移和旋转 RPE 分别恶化约 `5.57` 倍和 `5.31` 倍。

轨迹保存格式、四元数顺序、评估器坐标约定、图像清单和第一阶段缓存已经排除。
退化发生在回环测量构建和全局优化阶段。

## 2. 根因

### 2.1 坐标定义

定义：

- `W_i`：窗口 `i` 的原始 Pi3 局部世界坐标系；
- `W_0`：第一阶段采用的全局坐标系；
- `J`：一次 A、B 双侧联合 Pi3 推理产生的联合世界坐标系；
- `G_i = sim3_abs[i]`：从 `W_i` 到 `W_0` 的第一阶段绝对变换；
- `X_A`：从 `J` 到 `W_A` 的回环侧 A 局部对齐；
- `X_B`：从 `J` 到 `W_B` 的回环侧 B 局部对齐。

第一阶段已经把缓存中的 `camera_poses` 传播到 `W_0`。因此，联合 Pi3
预测与矫正缓存配准后得到的变换不是 `X_A` 和 `X_B`，而是：

```text
L_A = G_A compose X_A
L_B = G_B compose X_B
```

其中 `L_A` 和 `L_B` 都把联合坐标系 `J` 映射到第一阶段全局坐标系
`W_0`。

### 2.2 当前错误

当前设计和代码直接使用：

```text
C_global = L_B compose inverse(L_A)
```

`C_global` 表示两个全局对齐结果之间的漂移修正，但现有优化器要求的回环测量
必须把 `W_A` 映射到 `W_B`。把 `C_global` 直接传给优化器，混合了全局修正
和窗口局部相对位姿两种不同的坐标语义。

### 2.3 优化器要求

对于优化器节点 `G_A` 和 `G_B`，现有残差为：

```text
residual = Log(
    C_AB
    compose inverse(G_A)
    compose G_B
)
```

零残差要求 `C_AB` 是从 `W_A` 到 `W_B` 的相对测量。

联合 Pi3 实际提供的该测量是：

```text
C_AB = X_B compose inverse(X_A)
```

将 `X_A = inverse(G_A) compose L_A` 和
`X_B = inverse(G_B) compose L_B` 代入，得到：

```text
C_AB =
    inverse(G_B)
    compose L_B
    compose inverse(L_A)
    compose G_A
```

这才是可以交给现有 `Sim3LoopOptimizer` 的回环约束。

## 3. 设计选择

### 3.1 采用方案

本轮采用最小坐标语义修复：

1. 保持第一阶段矫正缓存和 `sim3_abs`、`sim3_edge` 契约不变；
2. 保持现有优化器变量、残差、求解器、阻尼和收敛逻辑不变；
3. 将全局对齐结果显式转换为窗口局部相对回环测量；
4. 增加可复现的回环候选和约束诊断输出；
5. 不在本轮引入未经真实数据标定的几何阈值。

### 3.2 不采用的方案

本轮不直接优化窗口修正量 `D_i`。该方案需要重写优化器变量和残差，改动范围
超过当前坐标错误的最小修复范围。

本轮也不增加尺度、旋转、位移或残差阈值。阈值需要先通过真实序列诊断数据
标定，否则可能把真实回环误判为异常。

## 4. 代码边界

### 4.1 回环测量转换

在回环模块提供一个单一职责的转换函数。输入为：

```python
sim3_abs_a
sim3_abs_b
alignment_a
alignment_b
```

输出为：

```python
loop_constraint_ab
```

函数只执行以下组合：

```text
global_correction =
    alignment_b compose inverse(alignment_a)

loop_constraint_ab =
    inverse(sim3_abs_b)
    compose global_correction
    compose sim3_abs_a
```

`LoopClosureEngine.process_loops()` 必须通过该函数构造约束，不能再把
`L_B compose inverse(L_A)` 直接送入优化器。

### 4.2 诊断输出

诊断只增加可观测性，不改变候选接受条件：

- 保存 SALAD 接受的帧对和相似度；
- 输出帧对映射后的窗口索引；
- 输出每条局部回环测量的尺度、旋转角和位移范数；
- 输出每条回环约束在第一阶段图上的初始 Sim(3) 残差；
- 输出顺序约束数量和回环约束数量。

相似度不能在 `get_loop_list()` 后永久丢失。磁盘诊断文件仍使用现有
`loop_closures.txt`，不新增结果目录层级。

### 4.3 保持不变

以下部分不属于本轮改动：

- 图像发现和自然排序；
- 统一的前 `30%` 配准置信度选择；
- 三种分割方法；
- 深度细化和锚点传播；
- SALAD 相似度阈值、`top_k` 和 NMS；
- 联合 Pi3 的图像范围；
- Sim(3) 优化器数学形式和边权重；
- 最终增量应用与聚合方式；
- 输出轨迹格式。

## 5. 异常处理

约束转换前必须验证四个 Sim(3) 的尺度、旋转和位移均为有限值，且尺度严格
大于零。无效约束应记录具体帧对和原因并跳过，不能进入优化器。

本轮只拒绝数学上无效的 Sim(3)，不根据经验阈值拒绝数值较大的有限约束。

如果所有候选都因无效 Sim(3) 被跳过，则沿用现有无回环路径，返回第一阶段
`sim3_abs`，不调用优化器。

## 6. 测试设计

### 6.1 非单位坐标变换

构造具有非单位尺度、非零旋转和非零位移的：

```text
G_A, G_B, X_A, X_B
```

再构造：

```text
L_A = G_A compose X_A
L_B = G_B compose X_B
```

转换函数必须恢复：

```text
X_B compose inverse(X_A)
```

该测试在旧实现上必须失败，用于证明测试确实覆盖云端暴露的坐标错误。

### 6.2 优化器零残差不变量

当节点绝对变换与回环测量一致时，验证：

```text
C_AB compose inverse(G_A) compose G_B == identity
```

测试必须使用非单位、非共线的 Sim(3)，不能只使用单位矩阵或纯尺度。

### 6.3 引擎约束集成

为 `LoopClosureEngine.process_loops()` 提供确定的 `L_A`、`L_B` 和缓存
`G_A`、`G_B`，验证传入优化器前保存的约束已经完成局部坐标转换。

### 6.4 诊断和回退

验证：

- `loop_closures.txt` 保留帧索引和相似度；
- 非有限或非正尺度的约束被跳过；
- 所有约束无效时不调用优化器；
- 无回环路径保持第一阶段绝对变换不变。

### 6.5 完整回归

运行全部测试，确保分割、第一阶段传播、缓存聚合、无回环和输出格式没有回归。

## 7. 验收标准

本地验收必须满足：

1. 坐标转换回归测试经历明确的红灯和绿灯；
2. 非单位 Sim(3) 测试通过；
3. 无回环行为测试通过；
4. 全部自动化测试通过；
5. 代码和设计中不再把全局修正量直接描述为窗口局部回环测量；
6. 未修改分割、锚点传播和 `0.3` 置信度逻辑。

云端验收使用现有 KITTI 00 第一阶段缓存重新构造回环约束和轨迹。必须同时
保存每条回环的诊断信息，并与 ATE `18.323618` 的优化前基线比较。真实序列
ATE 是否改善属于云端实验结果，不能由本地单元测试预先保证。
